# rulegen

Auto-generate `CLAUDE.md` and `.cursorrules` from codebase analysis. No LLM required.

Point it at any repo and get a structured context file your AI coding assistant can actually use.

## What it detects

- **Tech stack** — languages, frameworks, libraries, tools (from `package.json`, `pyproject.toml`, `go.mod`, `Cargo.toml`, etc.)
- **Project structure** — directory tree, monorepo layout, key files annotated
- **Conventions** — file naming style, test patterns, import style, TypeScript strictness
- **Common commands** — dev, build, test, lint scripts
- **What to avoid** — gitignore patterns, private packages

## Install

```bash
npm install -g rulegen
```

Or run directly without installing:

```bash
npx rulegen
```

## Usage

```bash
# Scan current directory, write CLAUDE.md
rulegen

# Scan a specific project
rulegen ./my-project

# Write .cursorrules instead
rulegen --format cursor

# Write both CLAUDE.md and .cursorrules
rulegen --format both

# Print to stdout (pipe-friendly)
rulegen --stdout

# Custom output path
rulegen --output docs/AI.md

# Overwrite existing file
rulegen --overwrite
```

## Example output

Given a Next.js + TypeScript + Prisma project, rulegen generates:

```markdown
# CLAUDE.md — my-app

> Auto-generated by rulegen. Review and customize.

## Tech Stack

- Runtime: **Node.js**
- Package manager: **pnpm**
- Languages: TypeScript
- Frameworks/Libraries: Next.js, React, Prisma, Tailwind CSS, Zod
- Tools: ESLint, Prettier, Docker, GitHub Actions

## Common Commands

- `pnpm dev` — next dev
- `pnpm build` — next build
- `pnpm test` — vitest
- `pnpm lint` — eslint .

## Project Structure

\`\`\`
my-app/
├── app/
│   ├── api/
│   ├── (auth)/
│   └── layout.tsx
├── components/
├── lib/
├── prisma/
│   └── schema.prisma
└── package.json
\`\`\`

## Key Files

- `package.json` — Project manifest
- `prisma/schema.prisma` — DB schema
- `.env.example` — Env template

## Conventions

- File naming: kebab-case (e.g., my-component.ts)
- Tests: colocated with source (*.test.* / *.spec.*)
- Imports: ES Modules (import/export)
- TypeScript strict mode enabled
- ESLint configured
- Prettier configured

## What to Avoid

- Do not commit: .env, .next, node_modules, dist
- Do not commit .env files with secrets
```

## Supported stacks

| Ecosystem | Detection |
|-----------|-----------|
| Node.js | package.json, lock files |
| Python | requirements.txt, pyproject.toml, Pipfile |
| Go | go.mod |
| Rust | Cargo.toml |
| Ruby | Gemfile |
| Any | File extension counting, directory analysis |

Frameworks detected include: React, Next.js, Vue, Nuxt, Svelte, SvelteKit, Astro, Remix, Express, Fastify, NestJS, Django, FastAPI, Flask, Gin, Echo, Actix, Axum, Rails, and more.

## Philosophy

v1 is zero-dependency, zero-LLM. Pure static analysis. Fast, offline, works on any machine.

The output is a starting point. Review it, fill in the `## Notes` section with architecture decisions and gotchas that only a human would know. The generated context handles the mechanical stuff; you add the insight.
